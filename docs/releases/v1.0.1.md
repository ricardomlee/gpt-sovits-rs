# gpt-sovits-rs 1.0.1

Patch release focused on the first-run and release experience after `v1.0.0`.

## Install

Docker is the recommended path:

```bash
git clone https://github.com/ricardomlee/gpt-sovits-rs.git
cd gpt-sovits-rs

python3 -m venv .venv-models
. .venv-models/bin/activate
pip install -r requirements-models.txt
python prepare_models.py

cp .env.example .env
docker compose -f compose.cpu.yml up -d
```

CUDA users should set `CUDA_IMAGE_TAG` in `.env` first:

```text
latest-cuda-sm75  RTX 20
latest-cuda-sm86  RTX 30
latest-cuda-sm89  RTX 40
latest-cuda-sm90  H100
```

Then run:

```bash
docker compose -f compose.cuda.yml up -d
```

## Model Preparation

Models are not bundled with the release assets or Docker images. Run:

```bash
pip install -r requirements-models.txt
python prepare_models.py
```

Runtime inference does not require Python. Python is only used for model preparation or raw checkpoint conversion.

See `docs/MODELS.md` for local source model conversion and custom checkpoint notes.

## Validation

This release should be published only after:

- `cargo fmt --all -- --check`
- `cargo test --locked --features http-api`
- `cargo clippy --all-targets --features http-api -- -D warnings`
- `cargo build --release --locked --features http-api,mkl`
- `rhysd/actionlint` against `.github/workflows/*.yml`
- Docker Compose config rendering for CPU and CUDA templates
- GitHub Actions CI build of the Linux x86_64 binary artifact
- GitHub Actions CI build of the CPU Docker image
