# gpt-sovits-rs 1.0.0

First production-oriented release of the Rust GPT-SoVITS inference service.

## Install

Docker is the recommended path:

```bash
git clone https://github.com/ricardomlee/gpt-sovits-rs.git
cd gpt-sovits-rs

python3 -m venv .venv-models
. .venv-models/bin/activate
pip install -r requirements-models.txt
python prepare_models.py

cp .env.example .env
docker compose -f compose.cpu.yml up -d
```

CUDA users should set `CUDA_IMAGE_TAG` in `.env` first:

```text
latest-cuda-sm75  RTX 20
latest-cuda-sm86  RTX 30
latest-cuda-sm89  RTX 40
latest-cuda-sm90  H100
```

Then run:

```bash
docker compose -f compose.cuda.yml up -d
```

## Highlights

- Single `gpt-sovits` binary with CLI and HTTP API.
- CPU Docker image for `linux/amd64`.
- CUDA Docker images for common NVIDIA compute capabilities.
- Automatic model preparation with `prepare_models.py`.
- Chinese punctuation now matches the Python GPT-SoVITS frontend, so commas and sentence punctuation are visible to GPT/BERT alignment instead of being dropped.
- Default inference uses KV cache and CUDA Graph where supported.

## Model Preparation

Models are not bundled with the release assets or Docker images. Run:

```bash
pip install -r requirements-models.txt
python prepare_models.py
```

See `docs/MODELS.md` for local source model conversion and custom checkpoint notes.
