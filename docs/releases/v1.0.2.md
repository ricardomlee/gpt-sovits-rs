# gpt-sovits-rs 1.0.2

Patch release focused on code structure, inference-path cleanup, and release confidence after
`v1.0.1`.

## Install

Docker is the recommended path:

```bash
git clone https://github.com/ricardomlee/gpt-sovits-rs.git
cd gpt-sovits-rs

python3 -m venv .venv-models
. .venv-models/bin/activate
pip install -r requirements-models.txt
python prepare_models.py

cp .env.example .env
docker compose -f compose.cpu.yml up -d
```

CUDA users should set `CUDA_IMAGE_TAG` in `.env` first:

```text
latest-cuda-sm75  RTX 20
latest-cuda-sm86  RTX 30
latest-cuda-sm89  RTX 40
latest-cuda-sm90  H100
```

Then run:

```bash
docker compose -f compose.cuda.yml up -d
```

## Highlights

- Split the large CLI entry point into focused CLI, doctor, and server modules.
- Split HTTP request parsing, WAV response handling, and buffered streaming helpers.
- Split inference defaults, text splitting, reference audio loading, and speaker feature
  preparation into smaller modules.
- Centralized GPT decode backend selection so plain, KV cache, and CUDA graph routing are easier to
  reason about.
- Removed duplicated target feature preparation and buffered HTTP inference code.
- Added an opt-in real-model smoke test for release validation against local model and voice files.
- Expanded development docs with the real-model smoke test, release checklist, benchmark notes, and
  debugging commands.

## Model Preparation

Models are not bundled with the release assets or Docker images. Run:

```bash
pip install -r requirements-models.txt
python prepare_models.py
```

Runtime inference does not require Python. Python is only used for model preparation or raw checkpoint
conversion.

See `docs/MODELS.md` for local source model conversion and custom checkpoint notes.
